# 编码实现报告

## 1. 项目概述

本项目实现“中国工艺美术馆预约与导览服务平台”MVP。系统面向游客、博物馆管理员、志愿者/讲解员三类角色，支持展览浏览、藏品查看、门票预约、活动报名、馆内导览、志愿者查看活动报名情况以及后台内容管理。

## 2. 技术选型

后端采用 Django 6 + Django REST Framework，数据库使用 SQLite，适合课程作业的本地部署与演示。前端采用 Vue 3 单页应用，通过 Fetch API 调用 RESTful 接口。认证采用后端生成的 Bearer Token，降低跨域和 CSRF 配置复杂度。

## 3. 主要实现内容

用户账号模块实现游客注册、登录、退出和角色区分。数据库使用 Django User 保存账号基础信息，Profile 表保存手机号、真实姓名、角色、部门和服务区域。

展览与藏品模块实现公开展览列表、展览详情、关联藏品和藏品详情。游客无需登录即可浏览已发布展览，下架展览不会展示在游客端。

门票预约模块实现参观场次查询、创建预约、个人预约记录、取消预约和管理员预约数据查询。创建预约和取消预约均使用数据库事务，并同步更新场次已预约人数。

活动报名模块实现活动列表、活动详情、游客报名、个人活动报名记录。系统会检查活动状态、活动时间、人数上限和重复报名。

志愿者/讲解员模块实现查看自己负责的活动和对应报名名单。接口会校验活动与志愿者之间的关联关系。

导览模块实现展厅、推荐路线、文字导览和地图图片展示。

后台内容管理模块实现管理员对展览、藏品、活动和导览信息的增删改查，其中展览和活动删除采用“下架”方式，保证历史数据可追踪。

## 4. 数据库设计

核心数据表包括 Profile、AuthToken、Exhibition、CollectionItem、VisitSlot、Reservation、MuseumActivity、ActivityRegistration、GuideInfo 和 ActivityVolunteer。表关系与详细设计报告一致：展览关联多个藏品，参观场次关联多个预约，活动关联多个报名，活动与志愿者通过中间表关联。

## 5. 接口实现

接口路径基本沿用详细设计报告中的 RESTful API 设计，包括 `/api/register/`、`/api/login/`、`/api/exhibitions/`、`/api/reservations/`、`/api/activities/`、`/api/guides/`、`/api/admin/*` 和 `/api/volunteer/*` 等。

## 6. 前端实现

前端位于 `frontend/index.html`，采用单页工作台形式。顶部导航包括展览、预约、活动、导览、账号、后台和志愿者入口。不同角色登录后可进入对应功能区域。页面使用响应式布局，移动端和桌面端均可访问。

## 7. 测试与验证

已验证以下流程：

- 获取展览列表
- 游客登录
- 查询可预约场次
- 游客创建预约
- 游客查看个人预约
- 管理员查看预约数据
- 志愿者查看负责活动

## 8. 已知问题与后续扩展

MVP 阶段未实现真实支付、实名认证、线下闸机核验、实时地图导航和消息通知。活动报名暂不支持游客自行取消。后续可扩展扫码入馆、语音导览、用户收藏评论、数据统计看板和 MySQL 部署。
